2. COMMITTED USER STORIES & ACCEPTANCE CRITERIA
--------------------------------------------------------------------------------

User Story 1 (US-01): General Destination Search
As a traveler, I want to search for travel destinations from the Home Page 
so I can quickly navigate to hotels and holiday packages.
* Acceptance Criteria:
  - The UI must have an input field for Destination, Dates, and Guests.
  - Clicking the search button should trigger a call to the backend search API.
  - The results should render cleanly on the page, showing both Hotels and Packages sections.

User Story 2 (US-02): Hotel Price Comparison
As a traveler, I want to compare hotel prices by destination and dates 
so I can find the best deal.
* Acceptance Criteria:
  - Users must be able to input Destination, Check-in date, Check-out date.
  - System must display a loading state while fetching hotel prices from the backend.
  - Results must display hotel image, title, location, rating, and best price from scraped sources (e.g., Booking.com, Agoda).

User Story 3 (US-03): AI-Powered Budget Search
As a budget-conscious traveler, I want to enter my total budget and get a combined 
flight+hotel package that fits within my limits.
* Acceptance Criteria:
  - The form must collect From City, Destination, Budget (INR), Travelers, Flight Date, Check-in, and Check-out dates.
  - The system must display a step-by-step progress indicator ("Connecting to flights", "Scanning hotels", etc.).
  - The final output must display flight and hotel cost summaries, ensuring total cost <= entered budget.
  - Results must show an "AI Selection Reason" (e.g., why this combo was chosen).


3. DESIGN APPROACH & UI/UML UPDATES
--------------------------------------------------------------------------------
- UI Design: We implemented a modern "Glassmorphism" UI using Tailwind CSS. 
  The dark mode aesthetic (Slate 950 deep backgrounds) with glowing gradients 
  (Indigo, Purple, Pink) was chosen to give a premium, AI-powered feel.
- Component Architecture (React):
  - Abstracted the layout into a reusable `MainLayout` component ensuring the 
    navigation bar and footer are consistent across all views.
  - Created distinct, stateful Page components (`Home.jsx`, `Hotels.jsx`, `BudgetSearch.jsx`).
- Data Mapping (UML context):
  - The client expects a robust JSON structure from the backend API carrying 
    specific nested objects (e.g., `budget search response` -> contains array of `combinations`, 
    each containing `flight` and `hotel` sub-objects).
